# 最終ゲート（出力前チェック）

## 整合

- 設計対象（UI/API/DB/Batch/NFR）と章構成が一致している
- 一覧（API/DB/Batch）があり、全項目に個別詳細がある
- 依存（外部IF/既存機能/既存DB）が明示されている

## 欠落が起きやすい箇所

- 認可/権限（UI/API）
- エラーと復旧（全般）
- 冪等性（API/Batch）
- 監査ログとPII（UI/API/DB）
- 既存変更の場合の互換/ロールバック

## まとめの必須項目

- リスク（代表）:
- 未決事項（誰が、いつ、何を決めるか）:
- 意思決定ログ（理由）:

## 不足情報が多い場合の停止基準

- 成功条件が不明
- 依存が不明
- 仕様の矛盾が解消できない

この場合は「確認質問（最大5件）」だけ出して停止する。
