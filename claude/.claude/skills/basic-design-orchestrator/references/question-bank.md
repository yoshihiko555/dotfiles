# 確認質問バンク（最大5件に絞る）

## 共通

- 今回の設計対象は UI/API/DB/Batch のどれですか？（複合なら“最終1枚で何を確定したいか”）
- 新規実装ですか？既存変更ですか？（既存変更なら現状の仕様/IF/DBをください）
- 成功条件（何ができたらOK）を1〜3個で教えてください
- 依存する外部システム/既存機能はありますか？
- 優先するNFR（性能/可用性/運用/セキュリティ）の順番は？
- 主要ユースケース（ユーザーがやりたい操作）を3〜5個挙げられますか？
- 境界（何が対象外か）を一言で言うと何ですか？

## UI

- 対象画面は単一画面ですか？画面群ですか？
- 主要な利用者ロール（権限）と、表示/操作制限は？
- 入力項目（必須・制約・初期値）を列挙できますか？
- エラー時の方針（どこで検証、どう表示、復旧手段）は？
- 画面から呼ぶAPI（あれば）とタイミングは？

## API

- APIの利用者（フロント/外部/バッチ）と認証方式は？
- API一覧（Method/Path/目的）を先に出せますか？
- 冪等性が必要なAPIはどれですか？
- エラーコード体系や既存の規約はありますか？
- 既存DBや外部IFへの依存はありますか？
- 重要フロー（成功系）を1つ、文章で説明できますか？（後でシーケンス図化）

## DB

- データの主体（エンティティ）を5〜10個で挙げられますか？
- 既存DBに追加/変更ですか？（マイグレーション制約は？）
- 検索や集計で重要なクエリは何ですか？（インデックス設計の根拠）
- 論理削除/監査カラム（created_at等）の方針は？
- PII/秘匿情報の扱い（暗号化/マスキング）は？
- データ整合性は強整合が必要ですか？最終的整合でも良いですか？

## Batch

- ジョブ一覧（目的・スケジュール・入力・出力）を出せますか？
- 再実行時に同じ結果である必要がありますか？（冪等性）
- 失敗時の運用（通知先・復旧手順・SLA）は？
- 処理量の目安（件数/日）と完了時間の目安は？
- 入力元/出力先（DB/ファイル/外部IF）は？
- 途中失敗した場合、どこまで進んだかを判定できますか？（再開ポイント）

## NFR

- SLO目安（レイテンシ/稼働率/RTO/RPO）はありますか？
- 監視したい指標とアラートの優先度は？
- ログに残すべき操作（監査対象）は？
- セキュリティ要件（規格/社内規程）があれば？
