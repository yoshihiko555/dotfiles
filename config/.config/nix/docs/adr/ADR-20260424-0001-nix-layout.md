# ADR-20260424-0001: Nix 設定ファイルのディレクトリ構成

- ステータス: 採用
- 決定日: 2026-04-24
- 関連: なし（Nix 関連初の ADR）

## 背景

Nix を段階的に導入していく方針（[ROADMAP](../ROADMAP.md) 参照）を決めた。
これまで実験用 flake は `sandbox/nix-minimal/` に置かれていたが、恒久配置ではない。
ROADMAP 上の Phase 1 以降で常用を始める前に、**後から移動が発生しない最終形の配置**を決めておく必要がある。

前提:

- dotfiles は `stow` ベース（`config/.config/*` → `~/.config/*` にリンク）
- `nix.conf` は Nix が **固定で `~/.config/nix/nix.conf` を読む**（パス変更不可）
- mozumasu/dotfiles が同種の構成で `.config/nix/` 配下に集約している

## 選択肢

### A. `config/.config/nix/` 配下に集約（採用）

- stow 配線（`stow config`）にそのまま乗る
- `nix.conf`（固定パス）と `flake.nix`（任意パス）を同居
- 将来 nix-darwin / home-manager に拡張してもディレクトリ移動なし

### B. トップ直下に `nix/` を新設

- 他の stow パッケージ（`shell/`, `claude/`）と同じ粒度で並ぶ
- ホームにはリンクされないため `nix.conf` が別管理になり、設定が分裂する
- `darwin-rebuild --flake` 前提の構成にしかなじまない

### C. `sandbox/nix-minimal/` をそのまま育てる

- 暫定感が残り、常用フェーズに入ると「一時検証」のシグナルと矛盾する
- 将来的に移動が発生することが確定する

## 決定

**A を採用。** 初期レイアウトは以下:

```
config/.config/nix/
├── flake.nix
├── flake.lock
├── nix.conf
├── README.md
└── docs/
    ├── ROADMAP.md
    └── adr/
        ├── DECISIONS.md
        └── ADR-20260424-0001-nix-layout.md
```

設定ファイルが増えてきた段階で、mozumasu 型（`hosts/` / `darwin/` / `home-manager/` / `modules/`）へ
**同じディレクトリ内で**拡張する。docs 配置は Nix ディレクトリ内に閉じる（全体横断の docs は切らない）。

## 影響

- `sandbox/nix-minimal/flake.nix` と `flake.lock` を `config/.config/nix/` へ `git mv`
- 空になった `sandbox/nix-minimal/` と `sandbox/` を削除
- `shell/.zprofile` に追加した Nix 初期化（login shell 対応）も本決定に連動するので同一コミットに含める
- `~/.config/nix/` は stow で自動リンクされるため、**手動配線は不要**（次回 `stow config` 実行時に有効化）

## 未確定事項（将来の ADR で扱う）

- Phase 1 で最初に `nix profile install` する CLI ツールの選定
- home-manager 採用可否（Phase 3）
- nix-darwin 採用可否（Phase 4）
