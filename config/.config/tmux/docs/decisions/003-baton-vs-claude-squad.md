# ADR-003: baton vs claude-squad の選定

- **状態**: 検討中
- **日付**: 2026-03-16
- **評価予定**: Phase 4 + Phase 7 (2026-03-23 までに判断)

## コンテキスト

AI エージェントのセッション管理ツールとして baton (自作 Go ツール) を使用中。
claude-squad は tmux ベースのセッション管理ツールで、代替候補。

## 比較

| 基準 | baton | claude-squad |
|------|-------|-------------|
| セッション管理 | tmux ステータスバーに表示 | 専用 TUI ダッシュボード |
| 状態検知 | JSONL パース (改善が必要) | tmux 統合 (ネイティブ) |
| カスタマイズ性 | Go で自由に改修可能 | `config.json` のみ |
| Codex/Gemini 対応 | 対応済み | プロファイルで対応 |
| git worktree | なし | 標準搭載 |
| 開発コスト | 自分で保守 | コミュニティ保守 |

## 決定

Phase 7 の試用期間で実際に使い比べて決定する。

## 評価基準

- 日常の Claude Code ワークフローとの統合度
- ai-orchestra の tmux-monitor との共存可否
- 操作の快適さとレスポンス
