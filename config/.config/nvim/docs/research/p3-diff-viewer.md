# Phase 3: Diff Viewer

## 背景

PR前のdiff確認やファイル履歴の確認をNeovim内で行いたい。
lazygitでもdiffは見られるが、文字レベルのハイライトや
サイドバイサイド表示で視認性の高いdiff体験が欲しい。

## 候補

| プラグイン | Stars | 特徴 | 備考 |
|-----------|-------|------|------|
| diffview.nvim | ~5,400 | 最も機能豊富（3-way merge、file history）| **メンテ停滞**（最終コミット 2024-06） |
| codediff.nvim | ~1,200 | VSCode風の文字レベルハイライト、C言語エンジン | **非常に活発**（2026-03 時点で毎日コミット） |

## 比較

| 項目 | diffview.nvim | codediff.nvim |
|------|---------------|---------------|
| diff精度 | 行レベル | **文字レベル（VSCode風）** |
| 表示モード | サイドバイサイド | サイドバイサイド + インライン切替 |
| マージ解消 | 3-way/4-way（最強） | 基本対応 |
| パフォーマンス | 良好（大規模repoで重い報告あり） | **C言語エンジン + 並列処理で高速** |
| メンテナンス | **停滞**（open issue 107件、応答なし） | **非常に活発** |
| PR風diff | `DiffviewOpen origin/main...HEAD` | `:CodeDiff main...` |
| ファイル履歴 | あり | あり |

## 結論

**codediff.nvim を採用**。

- 文字レベルのdiffハイライトが視覚的に優れている
- 活発なメンテナンスで安心感がある
- マージコンフリクト解消はlazygitで行う方針なので、diffview の 3-way 優位性は不要

→ ADR-20260313-010
