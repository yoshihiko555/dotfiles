# Phase 1: ファイルエクスプローラー選定

## 背景

プロジェクト構造の俯瞰とファイル操作をNeovim内で行うためのエクスプローラーを選定する。
fzf-luaが「名前で探す」担当なのに対し、エクスプローラーは「構造を見る」担当として導入する。

## 候補

| | neo-tree.nvim | oil.nvim | mini.files | nvim-tree.lua |
|---|---|---|---|---|
| UI | サイドバー/フローティング | バッファ | Miller列 | サイドバー |
| 依存 | nui + plenary + devicons | なし | なし | devicons |
| Git連携 | 組み込み | 要追加プラグイン | 要追加設定 | 組み込み |
| ファイル操作 | 専用キーマップ | Vim操作 | Vim操作 | 専用キーマップ |
| tokyonight | 専用ハイライト | 標準ハイライト | 専用ハイライト | 専用ハイライト |

## 比較

### neo-tree.nvim
- 強み: 機能最多、Git連携組み込み、tokyonight専用テーマ、LazyVimデフォルト
- 弱み: 依存3つ、設定項目が多い、専用キーマップの学習が必要

### oil.nvim
- 強み: Vim操作がそのままファイル操作、依存ゼロ、Vim学習との相乗効果
- 弱み: ツリー表示なし、Git連携は別途、見た目が地味

### mini.files
- 強み: Miller列が視覚的、依存ゼロ、バッファ編集式
- 弱み: Git連携なし、エコシステム小さい

### nvim-tree.lua
- 強み: 最も成熟・安定
- 弱み: neo-treeに世代交代されつつある

## 結論

neo-tree.nvimを採用。理由は → ADR にて記録。
