# Exercise 09: LSP ワークフロー
> 所要時間: 15-20分 | 前提: Exercise 08 (FzfLua)

## 目標
- LSP のジャンプ機能 (定義/参照/実装) を活用する
- Diagnostics を効率的に確認・移動する
- リネームとコードアクションを使う

## キーリファレンス

| キー | 説明 |
|------|------|
| `gd` | 定義へジャンプ |
| `gD` | 宣言へジャンプ |
| `gr` | 参照一覧 |
| `gi` | 実装へジャンプ |
| `K` | ホバー情報 |
| `<leader>rn` | リネーム |
| `<leader>ca` | コードアクション |
| `<leader>cd` | 行の Diagnostics |
| `[d` / `]d` | 前/次の Diagnostic |
| `<leader>fd` | Diagnostics (FzfLua) |
| `<leader>fs` | ドキュメントシンボル |
| `<leader>fw` | ワークスペースシンボル |
| `Ctrl+o` | ジャンプ前の位置に戻る |

**対応 LSP**: gopls, pyright, ts_ls

## 練習 1: 定義ジャンプと戻る

LSP 対応のプロジェクト (Go/Python/TypeScript) で実践しよう。

1. 任意の関数呼び出しにカーソルを置く
2. `gd` で定義元にジャンプ
3. `Ctrl+o` で元の位置に戻る
4. `gD` で宣言へジャンプ（言語による）

ポイント: `gd` → `Ctrl+o` のループが最も頻繁に使うパターン。

## 練習 2: 参照一覧と実装

1. 関数やメソッドの定義にカーソルを置く
2. `gr` で参照一覧を表示（FzfLua で表示される）
3. リストから選択して参照箇所にジャンプ
4. インターフェースのメソッドで `gi` → 実装一覧を表示

## 練習 3: ホバー情報

1. 関数名にカーソルを置いて `K` → 型情報やドキュメントが表示される
2. もう一度 `K` → ホバーウィンドウにフォーカスが移る（スクロール可能）
3. `q` でホバーを閉じる

## 練習 4: Diagnostics ナビゲーション

1. エラーや警告のあるファイルを開く（意図的にエラーを含むコードを書いてもよい）
2. `]d` で次の Diagnostic にジャンプ
3. `[d` で前の Diagnostic にジャンプ
4. `<leader>cd` で現在行の Diagnostic 詳細を表示
5. `<leader>fd` で全 Diagnostics を FzfLua で一覧表示

## 練習 5: リネーム

1. リネームしたい変数や関数にカーソルを置く
2. `<leader>rn` → 新しい名前を入力 → `Enter`
3. 全ての参照箇所が自動で更新されることを確認
4. `u` で Undo → 全ての変更が一度に戻る

## 練習 6: コードアクション

1. エラーや警告がある行にカーソルを置く
2. `<leader>ca` でコードアクション一覧を表示
3. 利用可能なアクション（import の自動追加、型の修正など）を選択

よくあるコードアクション:
- 不足している import の追加
- 未使用変数の削除
- インターフェースの自動実装

## 練習 7: シンボル検索

1. `<leader>fs` でドキュメント内のシンボル一覧を表示
2. 関数名やクラス名を入力して絞り込み
3. `<leader>fw` でワークスペース全体のシンボルを検索

## 確認テスト
- [ ] `gd` で定義にジャンプし `Ctrl+o` で戻れる
- [ ] `gr` で参照一覧を表示しジャンプできる
- [ ] `K` で型情報やドキュメントを確認できる
- [ ] `]d`/`[d` で Diagnostic 間を素早く移動できる
- [ ] `<leader>rn` でプロジェクト全体のリネームができる
- [ ] `<leader>ca` でコードアクションを実行できる
