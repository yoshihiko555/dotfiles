# Exercise 04: レジスタ・マーク
> 所要時間: 15-20分 | 前提: Exercise 01-03

## 目標
- 名前付きレジスタでコピー/ペーストを管理する
- `"0` レジスタで「削除に上書きされない yank」を活用する
- マークでファイル内の位置を記録・ジャンプする

## キーリファレンス

| キー | 説明 |
|------|------|
| `"{reg}y` | 指定レジスタにコピー |
| `"{reg}p` | 指定レジスタからペースト |
| `"0p` | 最後に yank したものをペースト |
| `"+y` | システムクリップボードにコピー |
| `"+p` | システムクリップボードからペースト |
| `:reg` | レジスタ一覧を表示 |
| `m{a-z}` | ローカルマークを設定 |
| `` `{mark} `` | マーク位置へジャンプ |
| `'{mark}` | マーク行頭へジャンプ |

## 練習 1: "0 レジスタ (yank 専用)

以下のテキストで、yank したものが削除で上書きされる問題を体験しよう。

```
コピーしたい行
ここにペーストしたい
コピーしたい行
削除する行B
ここにもペーストしたい
```

タスク:
1. `コピーしたい行` で `yy` (yank)
2. `削除する行A` で `dd` (削除) → この時点で `""` レジスタが削除内容で上書きされる
3. `ここにペーストしたい` の下で `p` → 削除した行がペーストされる（期待と違う）
4. `u` で戻し、代わりに `"0p` → yank した行がペーストされる

ポイント: `"0` は最後に `y` した内容を保持し、`d` で上書きされない。

## 練習 2: 名前付きレジスタ

以下の3つのテキストをそれぞれ別のレジスタに保存して使おう。

```
レジスタa用: Hello
レジスタb用: World
レジスタc用: Neovim
```

タスク:
1. `Hello` を `viw` で選択 → `"ay` でレジスタ `a` にコピー
2. `World` を `viw` で選択 → `"by` でレジスタ `b` にコピー
3. `Neovim` を `viw` で選択 → `"cy` でレジスタ `c` にコピー
4. `:reg abc` でレジスタの中身を確認
5. 以下の空欄を埋めよう:

```
 Hellofrom  Worldin Neovim!
```

`___` にカーソルを合わせて `ciw` → `Esc` → `"ap`, `"bp`, `"cp` をそれぞれ使う。

## 練習 3: システムクリップボード

```
このテキストをブラウザにペーストしよう
```

タスク:
1. 上の行を `V` で選択 → `"+y` でシステムクリップボードにコピー
2. ブラウザやターミナルで `Cmd+v` でペーストできることを確認
3. 逆に、ブラウザで何かをコピーして Neovim で `"+p` でペースト

## 練習 4: マーク

以下の長いテキストでマークを使ってジャンプしよう。

```
セクション1: 概要
ここに概要テキストがあります。
複数行にわたる説明文です。
とても長い文書だと想像してください。

セクション2: 詳細
ここに詳細テキストがあります。
技術的な説明が続きます。
まだまだ続く文書です。

セクション3: まとめ
結論をここに書きます。
最終的なまとめです。
```

タスク:
1. `セクション1` の行で `ma` → マーク `a` を設定
2. `セクション2` の行で `mb` → マーク `b` を設定
3. `セクション3` の行で `mc` → マーク `c` を設定
4. `` `a `` でセクション1へジャンプ
5. `` `c `` でセクション3へジャンプ
6. `` `b `` でセクション2へジャンプ
7. `` `. `` で最後に編集した位置へジャンプ

## 確認テスト
- [x] `"0p` で yank 内容を削除後もペーストできる
- [x] `"ay`, `"ap` で名前付きレジスタを使える
- [x] `"+y`/`"+p` でシステムクリップボードと連携できる
- [x] `ma` でマークを設定し `` `a `` でジャンプできる
- [x] `:reg` でレジスタ一覧を確認できる
