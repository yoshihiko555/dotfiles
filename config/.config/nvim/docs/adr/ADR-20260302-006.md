# ADR-20260302-006: ファイルエクスプローラーにneo-tree.nvimを採用

- **Phase**: 1
- **Status**: accepted
- **Date**: 2026-03-02

## Context

プロジェクト構造の俯瞰とファイル操作をNeovim内で行うためのエクスプローラーを選定する。
fzf-luaが「名前で探す」担当に対し、エクスプローラーは「構造を見る」担当として導入する。

## Decision

**neo-tree.nvim** を採用する。

理由:
- tokyonight専用ハイライトグループでGitステータスまで配色される（見た目の統一感）
- サイドバーUIはVSCodeからの移行に馴染みやすい
- Git連携が組み込みで、変更ファイルのステータスがツリー上で確認できる
- LazyVimデフォルトで情報・設定例が豊富

設定:
- `<leader>e` でサイドバー開閉、`<leader>E` で現在ファイルをreveal
- dotfiles・gitignored含め全ファイル表示
- neo-tree内の`<space>`マッピングを無効化（leaderキーとの競合防止）

キーマップ調整:
- LSPの `<leader>e`（Line diagnostics）→ `<leader>cd` に移動

## Alternatives

- **oil.nvim**: Vim操作でファイル管理でき学習効果が高いが、見た目が地味でツリー表示なし
- **mini.files**: Miller列表示が視覚的だが、Git連携なし
- **nvim-tree.lua**: 成熟・安定だがneo-treeに世代交代されつつある

## Consequences

- 依存が3つ（nui.nvim, plenary.nvim, nvim-web-devicons）と重め
- `<leader>e` をExplorerに割り当てたため、LSP Line diagnosticsは `<leader>cd` に移動
- which-keyにCodeグループ（`<leader>c`）を追加し、LSP系キーマップの整理を開始
