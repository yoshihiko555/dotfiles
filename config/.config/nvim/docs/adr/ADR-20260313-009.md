# ADR-20260313-009: Git ガターサインに gitsigns.nvim を採用

- **Phase**: 3
- **Status**: accepted
- **Date**: 2026-03-13

## Context

エディタ内でGitの変更状態をリアルタイムに可視化し、hunk操作やblameをNeovim内で行いたい。
Git操作自体はlazygitに委ねる方針のため、エディタ側は「見る・確認する」に特化する。

## Decision

**gitsigns.nvim** を採用する。

理由:
- inline blame・word-level diff・hunk stage/unstage を単一プラグインで全カバー
- Pure Lua、外部依存ゼロ、Neovim >= 0.9.0
- デファクト標準（~6,600 stars）で情報量が豊富
- lualine / trouble.nvim 等との連携実績が豊富

設定:
- `]c` / `[c` で hunk ナビゲーション（diff モードにも対応）
- `<leader>h` 系で hunk 操作（stage/reset/preview/blame）
- `<leader>tb` で inline blame トグル、`<leader>tw` で word diff トグル
- `ih` テキストオブジェクトで hunk 選択

## Alternatives

- **mini.diff**: 軽量だが inline blame がなく、別途プラグインが必要
- **vgit.nvim**: 全部入りだがスコープが広すぎ、plenary 依存
- **vim-gitgutter**: VimScript 製、モダン Neovim では選外

## Consequences

- `<leader>h` をGit hunkグループとして占有
- `<leader>t` にトグル系キーマップを配置（今後の拡張に注意）
- `]c` / `[c` はdiffモードのデフォルトと共存するよう条件分岐済み
