# ADR-20260302-004: ファジーファインダーにfzf-luaを採用

- **Phase**: 1
- **Status**: accepted
- **Date**: 2026-03-02

## Context

ファイル検索・grep・LSP連携などをNeovim内で完結させるためのファジーファインダーを選定する必要がある。

## Decision

**fzf-lua** を採用する。

理由:
- 検索速度が最速（fzfバイナリがGo製で高速なファジーマッチング）
- Lua依存ゼロで壊れにくい
- 組み込みpickerが70+で拡張なしでほぼ全ユースケースをカバー
- fzfバイナリがHomebrew経由で既にインストール済み（v0.67.0）
- LazyVimがデフォルトをtelescopeからfzf-luaに変更しており、エコシステムの勢いがある

## Alternatives

- **telescope.nvim**: エコシステム最大だが、plenary.nvim依存あり。デフォルトソーターが遅くfzf-native必須
- **mini.pick**: 依存ゼロだがpickerが少なくサードパーティ連携が乏しい
- **snacks.nvim picker**: folke製の新興picker。将来性はあるが成熟途中

## Consequences

- fzfバイナリが外部依存として必要（Homebrewで管理）
- telescope連携のみ提供するプラグインは利用できない可能性がある
- `<leader>f` プレフィックスでFind系操作、`<leader>g` でGit系操作を統一
